\doxysubsection{Detailed Description}
Framework to manage controls related to an object. 

A control is a mean to govern or influence the operation of an object, and in particular of a camera. Every control is defined by a unique numerical ID, a name string and the data type of the value it stores. The libcamera API defines a set of standard controls in the libcamera\+::controls namespace, as a set of instances of the Control class.

The main way for applications to interact with controls is through the Control\+List stored in the Request class\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Request\ *req\ =\ ...;}
\DoxyCodeLine{ControlList\ \&controls\ =\ req-\/>controls();}
\DoxyCodeLine{controls-\/>set(controls::AwbEnable,\ \textcolor{keyword}{false});}
\DoxyCodeLine{controls-\/>set(controls::ManualExposure,\ 1000);}
\DoxyCodeLine{}
\DoxyCodeLine{...}
\DoxyCodeLine{}
\DoxyCodeLine{int32\_t\ exposure\ =\ controls-\/>get(controls::ManualExposure);}

\end{DoxyCode}


The Control\+List\+::get() and Control\+List\+::set() functions automatically deduce the data type based on the control.
