\doxysubsection{Detailed Description}
Internal log message representation. 

The \doxylink{classlibcamera_1_1LogMessage}{Log\+Message} class models a single message in the log. It serves as a helper to provide the std\+::ostream API for logging, and must never be used directly. Use the \doxylink{libcamera_2base_2log_8h_a3577749fb48d57a158b8ac1a0b3ab57e}{LOG()} macro instead access the log infrastructure.

The move constructor is meant to support the \doxylink{namespacelibcamera_a1921244466953ec1310eb4194d6dcc1a}{\+\_\+log()} functions. Thanks to copy elision it will likely never be called, but C++11 only permits copy elision, it doesn\textquotesingle{}t enforce it unlike C++17. To avoid potential link errors depending on the compiler type and version, and optimization level, the move constructor is defined even if it will likely never be called, and ensures that the destructor of the {\itshape other} message will not output anything to the log by setting the severity to Log\+Invalid.
