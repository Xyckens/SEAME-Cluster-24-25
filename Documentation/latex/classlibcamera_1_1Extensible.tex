\doxysubsection{Detailed Description}
Base class to manage private data through a d-\/pointer. 

The \doxylink{classlibcamera_1_1Extensible}{Extensible} class provides a base class to implement the \href{https://wiki.qt.io/D-Pointer}{\texttt{ d-\/pointer}} design pattern (also known as \href{https://en.wikipedia.org/wiki/Opaque_pointer}{\texttt{ opaque pointer}} or \href{https://en.cppreference.com/w/cpp/language/pimpl}{\texttt{ p\+Impl idiom}}). It helps creating public classes that can be extended without breaking their ABI. Such classes store their private data in a separate private data object, referenced by a pointer in the public class (hence the name d-\/pointer).

Classes that follow this design pattern are referred herein as extensible classes. To be extensible, a class Public\+Class shall\+:


\begin{DoxyItemize}
\item inherit from the \doxylink{classlibcamera_1_1Extensible}{Extensible} class or from another extensible class
\item invoke the \doxylink{class_8h_a173cff214f6288d300840b7b474b7803}{LIBCAMERA\+\_\+\+DECLARE\+\_\+\+PRIVATE()} macro at the very top of the class definition
\item define a private data class named Public\+Class\+::\+Private that inherits from the \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} data class of the base class
\item invoke the \doxylink{class_8h_af850b1c484bfa1f484a53c6c2ac7fbad}{LIBCAMERA\+\_\+\+DECLARE\+\_\+\+PUBLIC()} macro at the very top of the \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} data class definition
\item pass a pointer to a newly allocated \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} data object to the constructor of the base class
\end{DoxyItemize}

Additionally, if the Public\+Class is not final, it shall expose one or more constructors that takes a pointer to a \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} data instance, to be used by derived classes.

The \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} class is fully opaque to users of the libcamera public API. Internally, it can be kept private to the implementation of Public\+Class, or be exposed to other classes. In the latter case, the members of the \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} class need to be qualified with appropriate access specifiers. The Public\+Class and \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} classes always have full access to each other\textquotesingle{}s protected and private members.

The Public\+Class exposes its \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} data pointer through the \doxylink{classlibcamera_1_1Extensible_a30a918eee516b7e1d97a7f346ce99fca}{\+\_\+d()} function. In the other direction, the pointer to the Public\+Class can be retrieved in functions of the \doxylink{classlibcamera_1_1Extensible_1_1Private}{Private} class using the \doxylink{class_8h_ad8c78641ec06c49edd227b63cc8449b5}{LIBCAMERA\+\_\+\+O\+\_\+\+PTR()} macro.
