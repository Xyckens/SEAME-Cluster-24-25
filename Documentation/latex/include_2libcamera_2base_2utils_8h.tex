\begin{DoxyParams}[1]{Parameters}

Range-\/based for loops are handy and widely preferred in C++, but are limited in their ability to replace for loops that require access to a loop counter. The enumerate() function solves this problem by wrapping the {\itshape iterable} in an adapter that, when used as a range-\/expression, will provide iterators whose value\+\_\+type is a pair of index and value reference.

The iterable must support std\+::begin() and std\+::end(). This includes all containers provided by the standard C++ library, as well as C-\/style arrays.

A typical usage pattern would use structured binding to store the index and value in two separate variables\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{std::vector<int>\ values\ =\ ...;}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keywordflow}{for}\ (\textcolor{keyword}{auto}\ [index,\ value]\ :\ utils::\mbox{\hyperlink{include_2libcamera_2base_2utils_8h_aae904c8027790ede4383b7b1d05ae279}{enumerate}}(values))\ \{}
\DoxyCodeLine{\ \ \ \ ...}
\DoxyCodeLine{\}}

\end{DoxyCode}


Note that the argument to enumerate() has to be an lvalue, as the lifetime of any rvalue would not be extended to the whole for loop. The compiler will complain if an rvalue is passed to the function, in which case it should be stored in a local variable before the loop.
