\doxysubsection{Detailed Description}
Create and manage cameras based on a set of media devices. 

The \doxylink{classlibcamera_1_1PipelineHandler}{Pipeline\+Handler} matches the media devices provided by a \doxylink{classlibcamera_1_1DeviceEnumerator}{Device\+Enumerator} with the pipelines it supports and creates corresponding \doxylink{classlibcamera_1_1Camera}{Camera} devices.

Pipeline handler instances are reference-\/counted through std\+::shared\+\_\+ptr$<$$>$. They implement std\+::enable\+\_\+shared\+\_\+from\+\_\+this$<$$>$ in order to create new std\+::shared\+\_\+ptr$<$$>$ in code paths originating from member functions of the \doxylink{classlibcamera_1_1PipelineHandler}{Pipeline\+Handler} class where only the \textquotesingle{}this\textquotesingle{} pointer is available.

Acquire exclusive access to the pipeline handler for the process. 

This function locks all the media devices used by the pipeline to ensure that no other process can access them concurrently.

Access to a pipeline handler may be acquired recursively from within the same process. Every successful \doxylink{classlibcamera_1_1PipelineHandler_a57ddfa7d422aefd4f99a382d6d711ffa}{acquire()} call shall be matched with a \doxylink{classlibcamera_1_1PipelineHandler_aead119b03920f468fa3bd091bfd65e5b}{release()} call. This allows concurrent access to the same pipeline handler from different cameras within the same process.

Pipeline handlers shall not call this function directly as the \doxylink{classlibcamera_1_1Camera}{Camera} class handles access internally.

\textbackslash{}context This function is called from the \doxylink{classlibcamera_1_1CameraManager}{Camera\+Manager} thread.

\begin{DoxyReturn}{Returns}
True if the pipeline handler was acquired, false if another process has already acquired it 
\end{DoxyReturn}

Pipeline handlers may override this in order to get resources such as opening devices and allocating buffers when a camera is acquired.

This is used by the uvcvideo pipeline handler to delay opening /dev/video\# until the camera is acquired to avoid excess power consumption. The delayed opening of /dev/video\# is a special case because the kernel uvcvideo driver powers on the USB device as soon as /dev/video\# is opened. This behavior should {\itshape not} be copied by other pipeline handlers.
