{\footnotesize\ttfamily libcamera\+::\+Stream\+Configuration\+::color\+Space}



The \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} for this stream. 

This field allows a \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} to be selected for this \doxylink{classlibcamera_1_1Stream}{Stream}.

The field is optional and an application can choose to leave it unset. Platforms that support the use of color spaces may provide default values through the generate\+Configuration() method. An application can override these when necessary.

If a specific \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} is requested but the \doxylink{classlibcamera_1_1Camera}{Camera} cannot deliver it, then the \doxylink{structlibcamera_1_1StreamConfiguration}{Stream\+Configuration} will be adjusted to a value that can be delivered. In this case the validate() method will indicate via its return value that the \doxylink{classlibcamera_1_1CameraConfiguration}{Camera\+Configuration} has been adjusted.

The frame\+Size value reports the number of bytes necessary to contain one frame of an image buffer for this stream. This total includes the bytes required for all image planes. {\footnotesize\ttfamily libcamera\+::\+Stream\+Configuration\+::stride}



\doxylink{classImage}{Image} stride for the stream, in bytes. 

The stride value reports the number of bytes between the beginning of successive lines in an image buffer for this stream. The value is valid after successfully validating the configuration with a call to \doxylink{classlibcamera_1_1CameraConfiguration_a29f8f263384c6149775b6011c7397093}{Camera\+Configuration\+::validate()}. For compressed formats (such as MJPEG), this value will be zero.
