\doxysubsection{Detailed Description}
Class to describe a color space. 

The \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} class defines the color primaries, the transfer function, the Y\textquotesingle{}Cb\+Cr encoding associated with the color space, and the range (sometimes also referred to as the quantisation) of the color space.

Certain combinations of these fields form well-\/known standard color spaces such as "{}s\+RGB"{} or "{}\+Rec709"{}.

In the strictest sense a "{}color space"{} formally only refers to the color primaries and white point. Here, however, the \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} class adopts the common broader usage that includes the transfer function, Y\textquotesingle{}Cb\+Cr encoding method and quantisation.

More information on color spaces is available in the V4\+L2 documentation, see in particular


\begin{DoxyItemize}
\item \href{https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/colorspaces-details.html\#col-srgb}{\texttt{ s\+RGB}}
\item \href{https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/colorspaces-details.html\#col-jpeg}{\texttt{ JPEG}}
\item \href{https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/colorspaces-details.html\#col-smpte-170m}{\texttt{ SMPTE 170M}}
\item \href{https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/colorspaces-details.html\#col-rec709}{\texttt{ Rec.\+709}}
\item \href{https://www.kernel.org/doc/html/latest/userspace-api/media/v4l/colorspaces-details.html\#col-bt2020}{\texttt{ Rec.\+2020}}
\end{DoxyItemize}

Note that there is no guarantee of a 1\+:1 mapping between color space names and definitions in libcamera and V4\+L2. Two notable differences are


\begin{DoxyItemize}
\item The s\+RGB libcamera color space is defined for RGB formats only with no Y\textquotesingle{}Cb\+Cr encoding and a full quantization range, while the V4\+L2 SRGB color space has a Y\textquotesingle{}Cb\+Cr encoding and a limited quantization range.
\item The s\+YCC libcamera color space is called JPEG in V4\+L2 due to historical reasons.

\end{DoxyItemize}

Adjust the color space to match a pixel format. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em format} & The pixel format\\
\hline
\end{DoxyParams}
Not all combinations of pixel formats and color spaces make sense. For instance, nobody uses a limited quantization range with raw Bayer formats, and the \doxylink{classlibcamera_1_1ColorSpace_a6c38886cf640948d04502d365d9e0ec9a6adf97f83acf6453d4a6a4b1070f3754}{Ycbcr\+Encoding\+::\+None} encoding isn\textquotesingle{}t valid for YUV formats. This function adjusts the \doxylink{classlibcamera_1_1ColorSpace}{Color\+Space} to make it compatible with the given {\itshape format}, by applying the following rules\+:


\begin{DoxyItemize}
\item The color space for RAW formats must be Raw.
\item The Y\textquotesingle{}Cb\+Cr encoding and quantization range for RGB formats must be \doxylink{classlibcamera_1_1ColorSpace_a6c38886cf640948d04502d365d9e0ec9a6adf97f83acf6453d4a6a4b1070f3754}{Ycbcr\+Encoding\+::\+None} and \doxylink{classlibcamera_1_1ColorSpace_ad3dd58d644ebcee4201886e4619f118fabbd47109890259c0127154db1af26c75}{Range\+::\+Full} respectively.
\item The Y\textquotesingle{}Cb\+Cr encoding for YUV formats must not be \doxylink{classlibcamera_1_1ColorSpace_a6c38886cf640948d04502d365d9e0ec9a6adf97f83acf6453d4a6a4b1070f3754}{Ycbcr\+Encoding\+::\+None}. The best encoding is in that case guessed based on the primaries and transfer function.
\end{DoxyItemize}
