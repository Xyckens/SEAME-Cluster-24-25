Registration of \doxylink{classlibcamera_1_1ConverterFactory}{Converter\+Factory} classes and creation of instances. 


\begin{DoxyParams}{Parameters}
{\em \+\_\+\+Converter} & The converter class type for this factory\\
\hline
\end{DoxyParams}
To facilitate discovery and instantiation of \doxylink{classlibcamera_1_1Converter}{Converter} classes, the \doxylink{classlibcamera_1_1ConverterFactory}{Converter\+Factory} class implements auto-\/registration of converter helpers. Each \doxylink{classlibcamera_1_1Converter}{Converter} subclass shall register itself using the \doxylink{converter_8h_a27079a1ce03e0a05e3a61890b08bb512}{REGISTER\+\_\+\+CONVERTER()} macro, which will create a corresponding instance of a \doxylink{classlibcamera_1_1ConverterFactory}{Converter\+Factory} subclass and register it with the static list of factories.

Creating an instance of the factory base registers it with the global list of factories, accessible through the \doxylink{classlibcamera_1_1ConverterFactoryBase_a4b8ef7e5270de11d84f05915bd8508c0}{factories()} function.

The factory {\itshape name} is used as unique identifier. If the converter implementation fully relies on a generic framework, the name should be the same as the framework. Otherwise, if the implementation is specialized, the factory name should match the driver name implementing the function.

The factory {\itshape compatibles} holds a list of driver names implementing a generic subsystem without any personalizations.
